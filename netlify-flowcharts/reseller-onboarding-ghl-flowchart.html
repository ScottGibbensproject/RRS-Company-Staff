Post-Cert Email Migration Strategy

Handling Existing vs New Resellers

Existing resellers have already received emails 1-18. Use dual triggers to avoid duplicate emails while maintaining a single unified workflow.

New Resellers

Trigger: Tag "rrs - direct agent" added

  • Start at Email 1
  • Receive all 18 original emails
  • After Email 18: Add tag "post-cert-emails-v1-complete"
  • Continue to Email 19+
Existing Resellers

Trigger: Tag "post-cert-continue" added

  • Skip directly to Email 19
  • Bulk apply "post-cert-continue" tag to existing resellers
  • They only receive new content
Workflow Structure
Triggers:
├─ Tag: "rrs - direct agent" → Entry Point A
└─ Tag: "post-cert-continue" → Entry Point B

Flow:
├─ Entry Point A (New)
│   └─ Email 1 → Email 2 → ... → Email 18
│       └─ Add tag: "post-cert-emails-v1-complete"
│
├─ Entry Point B (Existing) ──────────────────┐
│                                              │
└─ Email 19 → Email 20 → ... (New Content) ◄──┘

Agent Type Determination

How to Determine Agent Type After Certification?

Option 1: Feedback Form Field

Add a dropdown to the certification feedback form:

  • "I want to be a Direct Agent (work directly with RRS)"
  • "I am part of an ISO Group"
  • "I am a Referral Partner"

Recommended - Simple, immediate classification

Option 2: Initial Form Collection

Collect agent type on the initial inquiry form (Jotform/GHL form) before they start the course.

  • Store in custom field
  • Use field value in post-cert workflow

May need updates if they change intent
